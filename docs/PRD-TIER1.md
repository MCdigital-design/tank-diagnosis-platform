# PRD — Tier 1：浮盘健康监测三件套

> 版本 v0.1 · 2026-06-10 · 状态：**待评审**  
> 上游文档：`docs/ROADMAP.md`（2026-06-10 章节）

---

## 1. 背景与目标

行业专家清单（`docs/Message/Ideas.jpg`）消化后确认：浮盘（浮顶）专项监测是罐区业主价值最高、与本平台契合度最高的方向。现平台（v1.1.x 基线）已有倾斜、翻液监测，本期补齐浮盘健康闭环中价值最高的三块缺口：

1. 浮盘位置与状态机（高度 / 升降速率 / 落底 / 超高 / 超低）
2. 密封区温度 + 火灾级告警
3. 静电接地连续性监测

目标：在现有演示平台上跑通这三条监测链路（数据 → 3D / 面板展示 → 告警联动），形成可向客户演示的完整故事。

## 2. 范围

**范围内：**

- 上述三个功能，先用 mock 数据实现（沿用 `floatingRoofSensors.ts` 的造数模式）。
- 演示罐：储罐01、储罐02（对标 TG04 布局）。

**范围外：**

- 真实 DCS / SCADA 数据接入（接口形状保持兼容即可）。
- Tier 2 / Tier 3 项目（见路线图）。
- AI Copilot / MCP（另见 `docs/AI-BRAINSTORM.md`）。

## 3. 功能一：浮盘位置与状态机

### 3.1 数据模型

- 每罐一套浮盘行程数据：
  - `roofHeight`：浮盘当前高度，单位 m；
  - `travelRate`：升降速率，单位 mm/min；
  - 24h 高度曲线（与现有测点时序同结构）。
- 状态枚举（状态机，不只是数值阈值）：

| 状态 | 级别 |
|------|------|
| 正常漂浮 | 正常（绿） |
| 接近高限 | 预警（黄） |
| 超高位报警 | 报警（红） |
| 接近低限 | 预警（黄） |
| 超低位报警 | 报警（红） |
| 落底（高风险） | 高风险（深红 + 强调） |
| 升降速率异常 | 报警（红） |

### 3.2 UI

- 3D 场景：罐旁增加垂直行程标尺，标出当前高度与高 / 低限刻度。
- 罐信息卡：新增“浮盘状态”徽章，状态色绿 / 黄 / 红；落底为深红 + 闪烁或明显强调。
- 左 / 底部面板：显示当前高度、升降速率、距高限 / 低限余量。
- 状态变化进入告警列表，并与罐选中联动（沿用 `TankSelectionContext`）。

- [x] 演示数据中安排一罐处于“接近高限”，展示黄色状态。
- [x] 落底状态可在演示脚本中触发展示（`floatingRoofState.ts` 内 `DEMO_LANDED_TANK_ID`）。

## 4. 功能二：密封区温度 + 火灾级告警

### 4.1 需求

- 将现有 T4_x 温度环语义升级为“密封区温度”（一、二次密封圈温度监测）。
- 阈值三段保留（正常 / 预警 / 报警），新增第四级 **“火警”**（如 ≥ 某演示阈值）。
- 火警触发时：
  - 3D 测点红色脉冲强化；
  - 告警列表置顶，并标“火警”级；
  - 罐信息卡顶部出现红色横幅提示。
- `thresholdNote` 文案更新为密封区语义：一次 / 二次密封圈温度监测 · 火灾预警。

### 4.2 验收

- [x] 演示数据含一个火警级测点（可复用 T4_1）。
- [x] 告警列表点击火警条目，可跳转至该测点。

## 5. 功能三：静电接地连续性

### 5.1 数据模型

- `SensorType` 新增 `'grounding'`。
- 测点布局：浮盘静电线 N 条（如 4 条，环形内圈布点）+ 罐体接地 2 点。
- 状态为布尔：连接正常 / 断开报警；无连续数值曲线，时序为状态变化事件列表。

### 5.2 UI

- 3D 场景：用小型方形 / 菱形标记，区别于温度、液位的圆点。
- 罐信息卡设备清单表：新增“静电接地”行。
- 断开时进入告警列表。

### 5.3 验收

- [x] 演示数据含一条断开的静电线，完整展示报警链路（3D 标记 → 告警列表 → 罐卡片）。

## 6. 非功能要求

- 沿用现有架构：`floatingRoofSensors.ts` 模式扩展、`TankSelectionContext` 选中联动、告警-测点联动机制。
- 不引入新依赖。
- 保持 1920×1080 缩放布局与现有渲染性能。
- 中文文案风格与现有界面一致。
