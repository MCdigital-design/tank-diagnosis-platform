## 2026年6月10日 — 行业专家清单的消化分析与分级路线

来源：一位行业专家给出的罐区监测需求清单（`docs/Message/Ideas.jpg`），本节是对清单逐条消化后形成的结论，作为后续开发的依据。

### 一、专家清单五大部分逐条消化

#### 第一部分：内浮盘（浮顶）专项监测

专家在这部分着墨最多，也是罐区业主真实痛点所在。逐条对照现平台：

| # | 监测项 | 痛点 / 风险 | 现平台覆盖情况 |
|---|--------|-------------|----------------|
| 1 | 浮盘实时高度、升降速率 | 收发油时看不到浮盘是否正常移动；速率异常 = 卡盘或超速收发 | ❌ 未覆盖（只有液位百分比，无浮盘行程动态） |
| 2 | 落底、超高位 / 超低位报警 | 落底是高风险瞬态：浮盘下方形成气相空间 → 爆炸风险。需要离散状态机（漂浮 / 临界 / 落底），不只是数值 | ❌ 未覆盖 |
| 3 | 浮盘倾斜 | 倾斜过大 → 卡盘、沉盘前兆 | ✅ 已建（D4_x 倾角点 + 阈值） |
| 4 | 箱体进油（浮筒泄漏） | 浮力丧失 → 沉盘，且外部不可见 | ❌ 未覆盖 |
| 5 | 浮盘上部翻液 | 油品翻上浮盘 → 蒸发损耗、火灾风险 | 🟡 部分覆盖（D4_x 翻液 / 液位偏差点） |
| 6 | 密封间隙变化（弹性 / 舌形密封） | 间隙变大 → 油气损耗、火灾、环保问题 | ❌ 未覆盖，但与现有环形布点模型完全匹配 |
| 7 | 密封区域温度（火灾报警） | 一、二次密封圈火灾是浮顶罐最典型的火灾场景 | 🟡 接近：现有 T4_x 温度环可语义化为密封区温度，再增加火灾级告警 |
| 8 | 静电线连接状态 | 静电导线断开 → 点燃风险，目前靠人工巡检发现 | ❌ 未覆盖，但最容易补（布尔状态点） |

#### 第二部分：储罐本体监测

- **腐蚀 / 变形**：传感器类别不同、数据更新慢，UX 形态偏趋势报告而非实时告警，列为二期。
- **储罐静电导出**：与浮盘静电线同一模式，成本低，可与浮盘静电监测一并做。

#### 第三部分：油品日常取样与理化指标

密度、黏度、凝点、闪点、含水等指标，本质是 **LIMS / 化验数据的集成展示**，不是传感监测。后续在罐信息卡里做只读化验面板即可，不做取样硬件。

#### 第四部分：罐区设备巡检

泵、管线的外观、声音、振动监测，属于 CMMS / 泵预测性维护范畴，是成熟且拥挤的市场。**不自建**，未来走集成或合作。

#### 第五部分：现场环境 / 人员 / 作业合规

动火、受限空间、登高、临时用电等票证管理，劳保穿戴识别、闲杂人员、标识标牌检查。专家原话：“目前国内有人做，叫做智慧施工”。**明确不做**——技术栈（CV + 工作流）与本平台完全不同。对外口径：与智慧施工系统集成，不替代。

### 二、价值 × 契合度结论

| 清单部分 | 业主价值 | 与平台契合度 | 结论 |
|----------|----------|--------------|------|
| 一、浮盘专项监测 | 很高 | 极高 | **重点投入** |
| 二、罐体腐蚀 / 静电 | 中高 | 中 | 接地（静电导出）先做，腐蚀二期 |
| 三、油品理化指标 | 中 | 中低 | 展示级集成（只读面板） |
| 四、设备巡检 | 中 | 低 | 不做，走集成 |
| 五、HSE / 作业合规 | 高 | 极低 | 明确范围外，对接智慧施工 |

### 三、定位结论：不转向，收窄聚焦

**不做大而全的罐区管理软件，做浮盘（浮顶）健康监测的完整闭环：**

> 位置 / 升降 → 倾斜 / 翻液 → 密封 / 火灾 → 静电 → 箱体泄漏

配套能力：3D 可视化、可追溯阈值、AI 告警解释（AI / MCP 方向见 `docs/AI-BRAINSTORM.md`）。

### 四、分级路线

#### Tier 1 — 下一步开发（详见 `docs/PRD-TIER1.md`）

1. **浮盘位置与状态机**：高度、升降速率、落底 / 超高 / 超低报警。
2. **密封区温度环 + 火灾级告警语义**：基于现有 T4_x 温度环升级。
3. **静电接地连续性监测**：浮盘静电线 + 罐体接地。

#### Tier 2 — 下季度

4. 箱体（浮筒分舱）进油监测。
5. 密封间隙传感环。
6. 油品理化指标只读面板。

#### Tier 3 — 暂缓 / 范围外

7. 腐蚀变形趋势模块（待真实数据源）。
8. 设备巡检（集成对象，不自建）。
9. HSE / 作业票（范围外，集成智慧施工）。
